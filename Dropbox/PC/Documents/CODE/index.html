REPARER DES ERREURS

Je supprime une branche.

pour supprimer une branche, je tape :
git branch -d (nom de la branche)
Si la branche n'est pas vide, je tape : $ git branch -D (nom de la branche)
Ce qui supprimera également tous les fichiers qu'elle contient.

J'ai modifié une branche par erreur

L'erreur n'a pas été commit.
J'ai fait :
$ git add PremierFichier.txt
puis
$ git status
Je crée une REMISE avec :
$ git stash
Les modifications sont remisées pour revenir à une branche master propre.
Je vais enregistrer ces changements sur une autre branche.
Tout d'abord, je consulte la liste des remises :
$ git stash list
ensuite :
$ git stash apply stash@{(numéro)}

j'ai modifié une branche par erreur, et en plus réalisé le commit
Effacer une erreur qui a déjà été commit :

$ git log
me permet de récupérer la référence que je vais effacer
$ git reset --hard HEAD^
efface.
Je crée une nouvelle branche :
$ git branch branche2
je me positionne dessus :
git checkout branche2
puis : $ git reset --hard (8 premiers chiffres de la référence récupérée avec $ git log)
puis taper echap et :x

Je veux modifier le message de mon commit :

$ git commit --amend -m "ahahah"

J'ai oublié un fichier dans mon commit :

$ git commit --amend --no-edit
Cette commande permet d'ajouter des fichiers au dernier commit.

Je souhaite annuler un commit que j'ai pushé.

git revert permet de revenir à une version antérieure :
$ git revert HEAD^
